# Language: C

> The systems programming substrate — every major language either compiles to C, borrows from C syntax, or was designed in reaction to C's problems.

---

## Type System Snapshot

| Axis | C |
|------|---|
| Binding | Early (static dispatch always) |
| Typing | Static |
| Strength | **Weak** — implicit int↔pointer, silent truncation, sign coercions |
| Type system | Nominal (struct names; no inheritance) |
| Type inference | None — must annotate everything |
| Memory model | **Manual** — malloc/free; stack via automatic storage |

## Conceptual Map

### Compilation Pipeline

Every `.c` file passes through four distinct tools before becoming executable code. This is not abstracted away — understanding the pipeline explains `#include`, symbol errors, and `-D` flags.

```
  source.c        source.h
     │                │
     ▼                ▼
┌──────────────────────────────────────┐
│  PREPROCESSOR  (cpp / cc -E)         │
│  #include → paste text               │
│  #define → substitute tokens         │
│  #ifdef → strip excluded blocks      │
│  No type checking — pure text subs   │
└──────────────────┬───────────────────┘
                   │  preprocessed .c (pure C, no directives)
                   ▼
┌──────────────────────────────────────┐
│  COMPILER  (cc -S)                   │
│  Parse + type-check                  │
│  AST → IR → optimization passes      │
│  Undefined behavior exploited here   │
│  One .c → one .s                     │
└──────────────────┬───────────────────┘
                   │  assembly .s
                   ▼
┌──────────────────────────────────────┐
│  ASSEMBLER  (as)                     │
│  .s → machine code                   │
│  Symbols still unresolved            │
│  Produces relocatable object (.o)    │
└──────────────────┬───────────────────┘
                   │  object file(s) .o + libraries .a / .so
                   ▼
┌──────────────────────────────────────┐
│  LINKER  (ld, invoked by cc)         │
│  Resolve symbol references           │
│  Combine .o files + libc             │
│  Lay out text/data/bss segments      │
└──────────────────┬───────────────────┘
                   │
                   ▼
              executable
              (ELF / Mach-O / PE)
```

> **C# parallel**: MSBuild → Roslyn → IL → ngen/JIT is the analogous pipeline; the key difference is C has no runtime — the linker's output is the final artifact.

### Process Memory Layout

When the executable runs, the OS maps it into a virtual address space with four distinct regions. Pointers are just addresses into this space — knowing which region an address lives in tells you its lifetime and ownership rules.

```
High address
┌───────────────────────────────────────┐
│              STACK                    │
│  Local variables, function params     │
│  Return addresses, saved registers    │
│  Grows DOWNWARD ↓                     │
│  Lifetime: function call duration     │
│  Size: fixed (default ~1–8 MB)        │
├───────────────────────────────────────┤
│         (unmapped / grows)            │
├───────────────────────────────────────┤
│               HEAP                    │
│  malloc() / calloc() / realloc()      │
│  Grows UPWARD ↑                       │
│  Lifetime: explicit — malloc to free  │
│  You own it; forget free = leak       │
├───────────────────────────────────────┤
│          DATA SEGMENT                 │
│  .bss  — zero-initialized globals     │
│  .data — initialized globals/statics  │
│  Lifetime: program lifetime           │
│  static int x = 5;  lives here        │
├───────────────────────────────────────┤
│          CODE SEGMENT (.text)         │
│  Machine instructions                 │
│  Read-only (writing → segfault)       │
│  "hello" string literals live here    │
│  char *s = "hello"; → points here     │
└───────────────────────────────────────┘
Low address
```

**Region-to-declaration mapping:**

| Declaration | Region | Lifetime | Writable? |
|-------------|--------|----------|-----------|
| `int x;` inside function | Stack | Function call | Yes |
| `int *p = malloc(n)` | Heap | Until `free(p)` | Yes |
| `static int x = 0;` | Data (.bss/.data) | Program lifetime | Yes |
| `char *s = "hello";` | Code (.text) | Program lifetime | **No** — writing = UB |
| `char s[] = "hello";` | Stack (copy) | Function call | Yes |

## Syntax Reference Card

### Variables & Types
```c
int x = 5;              // declare + init
int x;                  // declare (indeterminate value on stack — UB to read before assign)
const int MAX = 100;    // compile-time constant
static int count = 0;   // static storage duration (lives for program lifetime)
int a, b, c;            // multiple declarations

// Integer types
char      c = 'A';      // typically 1 byte, may be signed or unsigned
short     s = 32767;
int       i = 2147483647;
long      l = 2147483647L;
long long ll = 9223372036854775807LL;
unsigned int u = 4294967295U;

// Fixed-width (prefer these)
#include <stdint.h>
int8_t  int16_t  int32_t  int64_t
uint8_t uint16_t uint32_t uint64_t

// Floats
float  f = 3.14f;
double d = 3.14;
```

### Equality & Comparison
```c
x == y      // value equality (primitives); pointer ADDRESS equality (pointers)
x != y
x < y  x > y  x <= y  x >= y

// Pointer comparison
int *p = &x, *q = &y;
p == q          // true only if same address (same variable)
*p == *q        // compare values they point to

// No .equals() — no objects
// memcmp(a, b, n) for byte-by-byte struct comparison
```

### Logical Operators
```c
&&    // logical AND  (short-circuit)
||    // logical OR   (short-circuit)
!     // logical NOT

&     // bitwise AND  (no short-circuit)
|     // bitwise OR   (no short-circuit)
^     // bitwise XOR
~     // bitwise NOT (complement)
<<    // left shift
>>    // right shift (arithmetic or logical depending on signedness)
```

### Collections
```c
// Array (fixed size, stack or static)
int a[] = {1, 2, 3};          // size inferred
int a[5] = {1, 2, 3};         // a[3]=0, a[4]=0 (zero-padded)
int a[3];                     // uninitialized — UB to read

// Struct
struct Point { int x; int y; };
struct Point p = {1, 2};      // positional init
struct Point p = {.x=1, .y=2}; // designated initializer (C99+)
p.x = 5;

// No built-in map, set, list — use library or implement
// Common: uthash.h, khash.h, stb_ds.h

// Dynamic array (heap)
int *arr = malloc(n * sizeof(int));
arr[i] = val;
free(arr);
```

### Control Flow
```c
// If
if (cond) {
    ...
} else if (cond2) {
    ...
} else {
    ...
}

// Ternary
int max = (a > b) ? a : b;

// Switch (fall-through by default!)
switch (x) {
    case 1:
        printf("one");
        break;       // without break, falls through
    case 2: case 3:  // multiple cases
        printf("two or three");
        break;
    default:
        printf("other");
}

// Loops
for (int i = 0; i < n; i++) { }
while (cond) { }
do { } while (cond);
```

### Strings & Characters
```c
// char: 1 byte — ASCII/extended ASCII
char c = 'A';           // single quotes
int code = (int)'A';    // 65

// String: null-terminated char array
char *s = "hello";            // string literal — READ ONLY (undefined if you write to it)
char s[] = "hello";           // copies into array — mutable
char s[10] = "hello";         // 10-byte buffer

// String operations (stdlib)
#include <string.h>
strlen(s)           // length (not including \0)
strcpy(dst, src)    // copy — dangerous (no bounds)
strncpy(dst, src, n)
strcat(dst, src)
strcmp(a, b)        // 0 if equal, <0 or >0 otherwise
strstr(haystack, needle)

// No interpolation — use printf/sprintf
printf("Hello, %s! Age: %d\n", name, age);
char buf[256];
snprintf(buf, sizeof(buf), "Hello, %s!", name);
```

### Null / Pointers
```c
#include <stddef.h>
NULL        // null pointer constant (expands to 0 or (void*)0)

int *p = NULL;
if (p != NULL) {    // always check before dereference
    *p = 5;
}
// Dereferencing NULL = segfault (undefined behavior)

// No Option type — pointers ARE your nullable type
// Return NULL to signal failure
int *find(int *arr, int n, int target) {
    for (int i = 0; i < n; i++)
        if (arr[i] == target) return &arr[i];
    return NULL;
}
```

### Functions
```c
// Declaration (prototype)
int add(int a, int b);

// Definition
int add(int a, int b) {
    return a + b;
}

// Function pointers (late binding mechanism)
int (*fn_ptr)(int, int) = &add;
fn_ptr(1, 2);   // call through pointer

// Variadic
#include <stdarg.h>
int sum(int count, ...) {
    va_list args;
    va_start(args, count);
    int total = 0;
    for (int i = 0; i < count; i++) total += va_arg(args, int);
    va_end(args);
    return total;
}
```

### Error Handling
```c
// Convention: return 0 or positive on success, negative on error
// errno (from <errno.h>) for system call errors

#include <errno.h>
FILE *f = fopen("file.txt", "r");
if (f == NULL) {
    perror("fopen");        // prints: "fopen: No such file or directory"
    fprintf(stderr, "Error %d: %s\n", errno, strerror(errno));
    return -1;
}

// Multiple return values via output parameters
int divide(int a, int b, int *result) {
    if (b == 0) return -1;    // error code
    *result = a / b;
    return 0;                  // success
}
```

## What Makes It Distinct

1. **No runtime** — compiled directly to machine code; no GC, no runtime type info, no bounds checking
2. **Undefined behavior (UB)** — a class of "the standard says anything can happen" errors: buffer overflow, signed integer overflow, null deref, use-after-free; compilers exploit UB for optimization
3. **Manual memory** — `malloc`/`free`; no destructor; forget to free = leak; double-free = UB
4. **Pointers are first-class** — pointer arithmetic, pointer to functions, pointer to pointers; this power is also the danger
5. **Preprocessor** — `#define`, `#include`, `#ifdef` — textual substitution that runs before compilation; not part of the type system

## Decision Cheat Sheet

| Decision | Use When | Avoid When / Watch Out For |
|----------|----------|---------------------------|
| **Stack allocation** (`int x;`, `int a[N];`) | Size known at compile time; lifetime is function-scoped; N is small (< a few KB) | Large arrays (stack overflow); lifetime must outlive current scope |
| **Heap allocation** (`malloc`) | Size determined at runtime; lifetime must outlive function; large buffers | Forgetting `free` (leak); double-free (UB); always check for NULL return |
| **`malloc(n)`** | Allocating uninitialized memory you will fully write before reading | Uninitialized reads are UB — use `calloc` if you need zero-init |
| **`calloc(count, size)`** | Zero-initialized array of count elements; integer overflow protection in size calc | Slower than malloc due to zeroing; still must `free` |
| **`realloc(ptr, newsize)`** | Growing/shrinking an existing heap block | May return a different pointer — always assign return value; passing a freed pointer = UB |
| **`free(ptr)`** | Every `malloc`/`calloc`/`realloc` must have exactly one `free` | Double-free = UB; `free(NULL)` is safe (no-op) |
| **Pointer** (`int *p`) | Nullable (can be NULL); pointer arithmetic needed; passing large structs by reference | Array decay: `int a[]` passed to function becomes `int *` — size lost |
| **Array** (`int a[N]`) | Fixed-size, stack-allocated collection; sizeof gives full size within declaring scope | Decays to pointer when passed to function — pass size separately |
| **`static` storage class** | Variable that persists across function calls (local static); file-scope variable hidden from other TUs | Local statics are not thread-safe (C11 `_Thread_local` for thread-local) |
| **`extern` storage class** | Declare a symbol defined in another translation unit | Not a definition — no storage allocated; must match type exactly |
| **`auto` storage class** | Default for local variables; rarely written explicitly | Redundant keyword in C (every local is auto by default) |
| **`signed` integers** | Arithmetic where negative values are meaningful | Overflow is **undefined behavior** — compiler may delete "impossible" checks |
| **`unsigned` integers** | Bit manipulation, sizes, indices, modular arithmetic | Underflow wraps silently: `0u - 1` = `UINT_MAX`; mixing signed/unsigned in comparisons promotes to unsigned |
| **C99 features** (`//` comments, `for` init, designated initializers, VLAs, `<stdint.h>`) | Default baseline for new code | VLAs are optional in C11+; avoid them |
| **C11 features** (`_Atomic`, `_Generic`, `_Static_assert`, threads) | Portable atomics, type-generic macros, compile-time assertions | VLAs made optional; `<threads.h>` not universally implemented |
| **C23 features** (`nullptr`, `bool`/`true`/`false` as keywords, `#embed`, `typeof`) | Modern compilers targeting latest standard | Compiler support varies; check target toolchain before adopting |

## Gotchas from C#

| C# behavior | C behavior | Consequence |
|-------------|------------|-------------|
| `string` is object, `==` compares value | `char*` comparison with `==` compares addresses | Silent logic bug |
| `int` is 32-bit always | `int` is "at least 16 bits" (usually 32) | Use `int32_t` |
| Arithmetic on `int` wraps in `checked {}` context | Signed overflow is **undefined behavior** | Silent wrong results |
| Array bounds checked | Array access unchecked | Buffer overflow → UB or exploit |
| `foreach` iterates safely | `for (int i = 0; i <= n; i++)` with off-by-one = UB | Off-by-one reads garbage |
| `null` is type-safe | `NULL` is `0` cast to pointer | Confuses int and pointer |
| GC manages memory | Must `free` every `malloc` | Memory leaks, double-free |

## When to Choose C

- Embedded systems and firmware (no runtime)
- Operating system kernels and drivers
- Performance-critical libraries that other languages FFI-call
- Anywhere executable size and startup time are hard constraints
- Legacy codebases

**Not for**: application development, web servers, data processing — unless extreme performance requirements.
