# Language: Go

> Simplicity as a design principle — a small language that says no to everything except what's needed for cloud-native services: structural interfaces, goroutines, and explicit error returns.

---

## Type System Snapshot

| Axis | Go |
|------|-----|
| Binding | Early + interface dispatch (structural) |
| Typing | Static |
| Strength | Strong — no implicit conversions |
| Type system | Structural for interfaces; nominal for everything else |
| Type inference | Partial — `:=` short declaration, function args |
| Memory model | GC (concurrent tri-color mark-sweep) |

## Syntax Reference Card

### Variables & Types
```go
// Short declaration (inside functions only — most common)
x := 5                  // type inferred: int
name := "Alice"
x, y := 1, 2            // multiple assignment

// Var declaration (package or function level)
var x int               // zero value: 0
var x int = 5
var x = 5               // inferred

// Constants
const MAX = 100         // untyped constant
const MAX int = 100     // typed constant
const (                 // iota enumeration
    A = iota            // 0
    B                   // 1
    C                   // 2
)

// Zero values — every type has a zero value (no uninitialized variables)
// int: 0, float: 0.0, bool: false, string: "", pointer: nil, struct: all zeros

// Integer types
int   int8   int16   int32   int64
uint  uint8  uint16  uint32  uint64
byte  = uint8        // alias
rune  = int32        // alias — Unicode code point
uintptr              // pointer-sized uint

float32  float64
complex64  complex128
bool
string                 // immutable, UTF-8 bytes
```

### Equality & Comparison
```go
// == works on all comparable types
x == y
x != y
x < y  x > y  x <= y  x >= y

// Comparable types: all basic types, pointers, channels, interfaces, arrays (not slices/maps!)
var a [3]int = [3]int{1,2,3}
var b [3]int = [3]int{1,2,3}
a == b          // true (arrays are comparable!)

// Slices are NOT comparable with ==
s1 := []int{1,2,3}
s2 := []int{1,2,3}
// s1 == s2  ERROR — invalid operation
reflect.DeepEqual(s1, s2)   // true (but slow, uses reflection)
slices.Equal(s1, s2)        // true (Go 1.21+ stdlib)

// Maps are NOT comparable with ==
// reflect.DeepEqual(m1, m2)  // works but slow

// Struct comparison (if all fields are comparable)
type Point struct { X, Y int }
Point{1,2} == Point{1,2}    // true

// Nil comparison
var p *int = nil
p == nil                    // true

// Interface comparison (compares type + value)
var i interface{} = 42
i == 42                     // true
```

### Logical Operators
```go
&&    // short-circuit AND
||    // short-circuit OR
!     // NOT

&     // bitwise AND
|     // bitwise OR
^     // bitwise XOR (also unary complement when used as !^)
&^    // AND NOT (bit clear) — unique to Go
<<    // left shift
>>    // right shift
```

### Collections
```go
// Array — fixed size (value type, not a pointer!)
var a [3]int
a := [3]int{1, 2, 3}
a := [...]int{1, 2, 3}      // size inferred
a[0]  len(a)
// Passing array to function copies it (use slice instead)

// Slice — dynamic, reference type (pointer + len + cap)
s := []int{1, 2, 3}
s := make([]int, 5)          // len=5, cap=5
s := make([]int, 0, 10)      // len=0, cap=10
s = append(s, 4)             // may reallocate
s = append(s, 4, 5, 6)
s = append(s, other...)       // spread
s[0]  len(s)  cap(s)
s[1:3]                        // slice of slice (shared backing array!)
copy(dst, src)                // copy elements

// Map — hash table
m := map[string]int{"a": 1, "b": 2}
m := make(map[string]int)
m["key"] = 1
v := m["key"]                 // zero value if key missing
v, ok := m["key"]             // ok is false if key missing
delete(m, "key")
for k, v := range m { }

// Struct
type Point struct {
    X, Y int                  // exported: uppercase
    name  string              // unexported: lowercase
}
p := Point{X: 1, Y: 2}
p := Point{1, 2}              // positional (avoid for clarity)
p.X = 5

// Pointer
p := &Point{1, 2}             // *Point
p.X                           // auto-deref (no need for p->X)
var p *Point = nil

// Channel
ch := make(chan int, 10)      // buffered
ch <- 1                       // send
v := <-ch                     // receive
close(ch)
```

### Control Flow
```go
// If — no parens, braces required
if x > 0 {
    ...
} else if x < 0 {
    ...
} else {
    ...
}

// If with initialization (scoped to the if block)
if err := doSomething(); err != nil {
    return err
}

if v, ok := m[key]; ok {
    use(v)
}

// No ternary operator in Go — write the if/else
var max int
if a > b {
    max = a
} else {
    max = b
}

// Switch — no fall-through by default
switch x {
case 1:
    ...
case 2, 3:          // multiple values
    ...
default:
    ...
}

// Switch with no condition (= if-else chain)
switch {
case x < 0:
    ...
case x == 0:
    ...
default:
    ...
}

// Type switch
switch v := x.(type) {
case int:
    fmt.Printf("int: %d\n", v)
case string:
    fmt.Printf("string: %s\n", v)
default:
    fmt.Printf("other: %T\n", v)
}

// For — the only loop keyword
for i := 0; i < 10; i++ { }         // C-style
for i < 10 { i++ }                  // while-style
for { }                              // infinite
for i, v := range slice { }         // range over slice
for k, v := range m { }             // range over map
for i, r := range str { }           // range over string (i=byte index, r=rune)
for v := range ch { }               // range over channel
```

### Strings & Characters
```go
// string — immutable sequence of bytes (UTF-8 encoded)
s := "hello"
s := "hello" + " " + "world"    // concatenation
s := `raw string
multi-line
no \n escaping`

len(s)                          // BYTE count, not rune count
utf8.RuneCountInString(s)       // Unicode code point count

// Byte vs rune
s[0]                            // byte (uint8)
for i, r := range s { }        // i = byte index, r = rune (int32)
[]byte(s)                       // convert to byte slice
[]rune(s)                       // convert to rune slice
string(65)                      // "A" (from int)

// rune (int32) — Unicode code point
r := 'A'                        // rune literal
r := '\U0001F60A'         // emoji rune literal (Unicode escape)
fmt.Printf("%c", r)            // print as character

// String formatting (no interpolation — use fmt)
s := fmt.Sprintf("Hello, %s! Age: %d", name, age)
s := fmt.Sprintf("%.2f", 3.14159)

// strings package
strings.Contains(s, "sub")
strings.HasPrefix(s, "prefix")
strings.ToUpper(s)  strings.ToLower(s)
strings.TrimSpace(s)
strings.Split(s, ",")           // []string
strings.Join([]string{...}, ",")
strings.Replace(s, "old", "new", n)  // n=-1 for all
strings.Builder{}               // efficient concatenation
```

### Null / Nil
```go
// nil — the zero value for pointer, interface, func, slice, map, channel
var p *int          // nil pointer
var s []int         // nil slice (len=0, cap=0 — usable!)
var m map[string]int // nil map (readable: m["k"]=0; WRITE panics)
var i interface{}   // nil interface
var f func()        // nil function

// Nil check
if p != nil { *p = 5 }
if s != nil { ... }     // often unnecessary — nil slice has len 0

// No Option<T> — use pointer or (T, bool) pattern
func findUser(id int) (*User, bool) {
    u, ok := db[id]
    return &u, ok
}
u, found := findUser(42)
if !found { return }

// Common nil-safe pattern
type Config struct { ... }
func (c *Config) Name() string {
    if c == nil { return "" }    // nil receiver is OK in Go
    return c.name
}
```

### Functions
```go
func add(a, b int) int {         // multiple params of same type
    return a + b
}

// Multiple return values
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}

result, err := divide(10, 2)
if err != nil {
    return fmt.Errorf("divide: %w", err)  // wrap error
}

// Named return values
func minMax(a, b int) (min, max int) {
    if a < b { return a, b }
    return b, a
}

// Variadic
func sum(nums ...int) int {
    total := 0
    for _, n := range nums { total += n }
    return total
}
sum(1, 2, 3, 4)
sum(nums...)    // spread slice

// First-class functions
f := func(x int) int { return x * 2 }
apply := func(f func(int) int, x int) int { return f(x) }

// Defer (cleanup — runs LIFO when function returns)
func readFile(path string) error {
    f, err := os.Open(path)
    if err != nil { return err }
    defer f.Close()             // guaranteed to run
    // ... use f
}

// Method
func (p Point) Distance() float64 {
    return math.Sqrt(float64(p.X*p.X + p.Y*p.Y))
}
func (p *Point) Scale(factor int) {
    p.X *= factor
    p.Y *= factor
}

// Interface — satisfied implicitly
type Stringer interface {
    String() string
}
// Any type with String() string method satisfies Stringer
```

### Error Handling
```go
// Go errors are values — always check them
result, err := someOperation()
if err != nil {
    return fmt.Errorf("context: %w", err)  // wrap with context
}

// Error wrapping and unwrapping (Go 1.13+)
var pathErr *os.PathError
if errors.As(err, &pathErr) {
    fmt.Println(pathErr.Path)
}

if errors.Is(err, os.ErrNotExist) {
    // file not found
}

// Custom errors
type AppError struct {
    Code    int
    Message string
}
func (e *AppError) Error() string {
    return fmt.Sprintf("error %d: %s", e.Code, e.Message)
}

// Panic/recover (for truly exceptional situations — not normal errors)
defer func() {
    if r := recover(); r != nil {
        log.Printf("recovered: %v", r)
    }
}()
panic("unrecoverable")
```

### Concurrency
```go
// Goroutines — lightweight threads (managed by Go runtime)
go func() {
    result := compute()
    ch <- result
}()

// Channels — communication between goroutines
ch := make(chan int)        // unbuffered — send blocks until receive
ch := make(chan int, 10)    // buffered — send blocks when full

// Select — multiplex channels
select {
case v := <-ch1:
    fmt.Println("from ch1:", v)
case ch2 <- x:
    fmt.Println("sent to ch2")
case <-time.After(1 * time.Second):
    fmt.Println("timeout")
default:
    fmt.Println("non-blocking")
}

// sync.WaitGroup
var wg sync.WaitGroup
for _, item := range items {
    wg.Add(1)
    go func(item Item) {
        defer wg.Done()
        process(item)
    }(item)
}
wg.Wait()
```

## What Makes It Distinct

1. **Structural interfaces — implicit satisfaction** — a type satisfies an interface just by having the right methods. No `implements` declaration. This enables retroactive interface implementation and reduces coupling.
2. **Goroutines + channels = CSP** — Communicating Sequential Processes. Goroutines are cheap (2KB stack, grows on demand). Channels are the idiomatic way to share state. "Share memory by communicating."
3. **Error as value** — Go has no exceptions. Every error-producing function returns `(T, error)`. You handle it or propagate it with `fmt.Errorf("context: %w", err)`. Verbose but explicit.
4. **No generics (until Go 1.18)** — Go resisted generics for 12 years. Now they exist but are simpler than Rust/Java. Type parameters with constraints.
5. **Small language surface** — 25 keywords. One way to do most things. Easy to read other people's Go code. The Go team famously says no to features.

## Gotchas from C#

| C# behavior | Go behavior | Consequence |
|-------------|-------------|-------------|
| `List<T>` is always a reference | Slice is a reference to backing array — sub-slices share backing | Modifying sub-slice modifies original |
| `Dictionary<K,V>` readable when null | Writing to nil map **panics** | Initialize with `make` |
| Classes have inheritance | Go has no inheritance — use composition (embedding) | Different OOP model |
| Multiple return not idiomatic | Multiple return values are idiomatic `(T, error)` | Learn to love it |
| `try/catch` | `if err != nil` on every call | More verbose; more explicit |
| `using` for cleanup | `defer` for cleanup | `defer` runs at function end, not block end |
| Ternary `cond ? a : b` | No ternary | Write the if/else |

## Decision Cheat Sheet

| Decision | Use X | When Y |
|----------|-------|--------|
| **goroutine vs `sync.WaitGroup` vs channel** | goroutine alone | Fire-and-forget; result not needed by caller |
| | goroutine + `WaitGroup` | Fan-out work; need to know when all are done; no inter-goroutine data |
| | goroutine + channel | Goroutines need to communicate results back; pipeline patterns |
| **pointer receiver vs value receiver** | pointer receiver `(p *T)` | Method mutates the struct; struct is large (avoid copy cost) |
| | value receiver `(p T)` | Read-only method; small struct; consistency when T has no pointer methods |
| **`errors.Is` vs `errors.As` vs type assertion** | `errors.Is(err, target)` | Check for a specific sentinel error value anywhere in the chain |
| | `errors.As(err, &target)` | Unwrap chain looking for a specific error type; extract its fields |
| | type assertion `err.(*MyErr)` | Direct type check without unwrapping; only use when not using `%w` wrapping |
| **`interface{}` / `any` vs generics** | `interface{}` / `any` | Pre-Go 1.18 code; truly heterogeneous containers; `fmt`-style variadic |
| | generics `[T constraint]` | Type-safe algorithms over multiple types; avoid `interface{}` + type assertion boilerplate |
| **`make` vs `new` vs composite literal** | `make(T, ...)` | Slices, maps, channels — initializes internal structure (len/cap/buffer) |
| | `new(T)` | Allocates zeroed T and returns `*T`; rarely needed; prefer composite literal |
| | `T{...}` composite literal | Structs, arrays, slices, maps with known values; most readable |
