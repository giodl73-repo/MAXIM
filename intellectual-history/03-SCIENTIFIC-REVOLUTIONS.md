# Scientific Revolutions: Kuhn and Critics

## The Big Picture

Thomas Kuhn's Structure of Scientific Revolutions (1962) is the most influential work in
20th-century philosophy/history of science. It introduced "paradigm," "normal science," and
"scientific revolution" into common use. Its reception generated a rich debate: Lakatos,
Feyerabend, Popper, and Laudan all responded.

```
+-------------------------------------------------------------------+
|              MODELS OF SCIENTIFIC CHANGE                          |
|                                                                   |
|  CLASSICAL (Cumulative)       KUHN (Revolutionary)               |
|  +---------------------+      +-------------------------+         |
|  | Science adds to a   |      | Normal science within   |         |
|  | growing body of     |      | paradigm -> anomalies   |         |
|  | knowledge           |      | -> crisis -> revolution |         |
|  | Errors corrected;   |      | Paradigm shift:         |         |
|  | truths accumulate   |      | incommensurable worlds  |         |
|  +---------------------+      +-------------------------+         |
|                                                                   |
|  POPPER (Falsificationism)    LAKATOS (Research Programmes)      |
|  +---------------------+      +-------------------------+         |
|  | Science progresses  |      | Hard core protected by  |         |
|  | by bold conjectures |      | auxiliary belt          |         |
|  | and ruthless        |      | Progressive vs.         |         |
|  | refutations         |      | degenerative programmes |         |
|  +---------------------+      +-------------------------+         |
|                                                                   |
|  FEYERABEND                   LAUDAN (Problem-Solving)           |
|  +---------------------+      +-------------------------+         |
|  | "Against Method":   |      | Science solves          |         |
```

## Kuhn: The Structure of Scientific Revolutions

### Paradigms

```
  PARADIGM (Kuhn's central and most contested concept):
  In narrow sense: an exemplary problem-solution that serves as model
  for subsequent work. (Kuhn later preferred "exemplar.")

  In broad sense (Masterman identified 21 senses): the entire constellation
  of beliefs, values, techniques, and puzzle-solutions shared by a community.
  (Kuhn later called this a "disciplinary matrix.")

  KEY PROPERTIES:
  1. Paradigms solve "paradigmatic" problems that define the field.
  2. They are never fully articulated — tacit knowledge is crucial.
  3. They are not "proven" before adoption; they are adopted because
     they open up productive research directions.

  EXAMPLES:
  Ptolemaic astronomy -> Copernican/Newtonian paradigm
  Phlogiston chemistry -> Lavoisier's oxygen chemistry
  Caloric theory of heat -> Thermodynamics
  Classical mechanics -> Quantum mechanics
  Newtonian gravity -> General Relativity
```

### Normal Science

```
  NORMAL SCIENCE: Research within an accepted paradigm.
  The paradigm defines which problems are legitimate,
  what counts as a solution, and what methods are appropriate.

  "Puzzle-solving": scientists solve pre-defined puzzles
  (not open-ended problems) within the paradigm's framework.

  WHY "PUZZLE" NOT "PROBLEM":
  A puzzle has a known solution; the challenge is to find it.
  Normal science assumes the paradigm is right and tries to
  make nature fit the paradigm's predictions.

  KEY FEATURE: Anomalies are NOT treated as refutations.
  When experiment disagrees with theory: first, check the experiment.
  Then, try auxiliary hypotheses. Only prolonged, serious anomalies
  lead to crisis.

  EXAMPLE: Le Verrier calculated where a planet had to be to account
  for anomalies in Uranus's orbit. The planet (Neptune) was found.
  Normal science success: don't abandon the paradigm, find the planet.
```

### Scientific Revolutions

```
  REVOLUTION STRUCTURE:

  Normal science
       |
       v  Accumulation of anomalies
  Crisis (anomalies resist resolution)
       |
       v  Proliferation of competing theories
  Revolution (paradigm replaced)
       |
       v  New normal science

  WHAT TRIGGERS REVOLUTION:
  Not a single anomaly, but a prolonged series that undermines
  confidence in the paradigm.
  New paradigm typically proposed by newcomers to the field,
  not established practitioners (who have too much invested).

  GESTALT SWITCH METAPHOR:
  The shift in seeing is sudden and total.
  Scientists who made the switch say the world looks different.
  Kuhn: "working in a different world" after the revolution.
```

### Incommensurability

```
  INCOMMENSURABILITY: Paradigms before and after a revolution cannot be
  directly compared on a common scale.

  TWO TYPES (Kuhn's later refinement):
  1. METHODOLOGICAL: Different criteria for what counts as good science.
     Newton vs. Aristotle used different notions of explanation.

  2. SEMANTIC: Key terms change meaning across paradigms.
     "Mass" means different things in Newtonian and Einsteinian physics.
     "Oxygen" in Lavoisier's chemistry is not the same concept as
     Priestley's "dephlogisticated air" even though they refer to the same element.

  CONSEQUENCE: Paradigms cannot be rationally compared by appeal to
  neutral, theory-independent evidence. Comparison requires translation,
  and perfect translation is impossible.

  CRITICS ARGUE:
  If paradigms are fully incommensurable, how does Kuhn himself compare
  Ptolemy and Copernicus? He must presuppose some common measure.
  KUHN'S REPLY: Partial incommensurability; translation is imperfect but not impossible.
```

## Karl Popper: Falsificationism

Popper is the foil and interlocutor Kuhn most directly challenges.

```
  POPPER (Logic of Scientific Discovery, 1934; Conjectures and Refutations, 1963):
  Science progresses by:
  1. Bold conjectures (theories that stick their necks out)
  2. Severe attempts at refutation (not confirmation)
  3. Falsification: if experiment contradicts theory, theory is refuted

  DEMARCATION CRITERION:
  Science vs. non-science: scientific theories are FALSIFIABLE.
  Astrology, Marxism (as practiced), psychoanalysis: not falsifiable, not scientific.

  KUHN'S REPLY:
  Normal science is NOT falsificationist.
  Scientists do not abandon paradigms when they conflict with data.
  Popper's ideal scientist doesn't exist; normal science is puzzle-solving.

  KUHN: The history of science doesn't match Popper's model.
  POPPER: Kuhn's description of normal science is accurate but unfortunate;
  it describes "normal" science but not GOOD science.
```

## Imre Lakatos: Scientific Research Programmes

```
  LAKATOS (Methodology of Scientific Research Programmes, 1978):
  A middle ground between Kuhn and Popper.

  SCIENTIFIC RESEARCH PROGRAMME:
  +-----------------------------------------------+
  |                   HARD CORE                   |
  |   Central assumptions; not to be refuted      |
  |   (by methodological fiat: negative heuristic)|
  +-----------------------------------------------+
  |               PROTECTIVE BELT                 |
  |   Auxiliary hypotheses; can be modified       |
  |   (positive heuristic: direct development)   |
  +-----------------------------------------------+
  |             POSITIVE HEURISTIC                |
  |   Programme's research agenda; what to do next|
  +-----------------------------------------------+

  HARD CORE:
  Newton's three laws + gravitational law.
  Cannot be refuted without abandoning the programme.

  PROTECTIVE BELT:
  Initial conditions, assumptions about air resistance, auxiliary masses.
  These absorb anomalies; Newtonian programme modified them, not core.

  PROGRESSIVE VS. DEGENERATIVE PROGRAMME:
  Progressive: each modification of protective belt predicts novel facts
  that are then confirmed (growing empirical content).
  Degenerative: modifications are ad hoc; no new predictions; just defending
  the hard core post hoc.
  Scientists should abandon degenerating programmes and join progressive ones.

  ADVANTAGE OVER KUHN:
  Provides a rational criterion for theory change (progressive vs. degenerative)
  without requiring a sudden gestalt switch.

  EXAMPLE:
  Bohr's atomic theory: hard core (quantized angular momentum),
  protective belt (selection rules, various atomic models).
  Programme was progressive: each new atomic spectrum explained,
  new predictions made and confirmed.
```

## Paul Feyerabend: Against Method

```
  FEYERABEND (Against Method, 1975):
  No single methodological rule governs all successful science.
  Every rule Popper, Kuhn, or Lakatos proposes has been violated
  by episodes in the history of science that everyone counts as progress.

  "ANYTHING GOES":
  Not a positive methodological prescription but a critical one.
  "Anything goes" is the only principle that doesn't hinder science.

  EXAMPLE: Galileo and Copernicus.
  Copernican theory was initially WORSE empirically than Ptolemy.
  Galileo violated standard methodological rules:
  - Used propaganda and rhetoric to persuade
  - Appealed to a telescope whose reliability was contested
  - Ignored strong (contemporary) arguments against Earth's motion
  Galileo succeeded through "counter-induction" (proposing the
  opposite of what the data supported) and rhetorical skill.

  IMPLICATION:
  Science is not more rational than other knowledge forms.
  Methodological enforcement would have killed science.

  CRITICS:
  "Anything goes" is a reductio, not a methodology.
  Feyerabend is more provocateur than constructive theorist.
  His reading of Galileo is contested by historians.
```

## Larry Laudan: Problem-Solving Model

```
  LAUDAN (Progress and Its Problems, 1977; Science and Relativism, 1990):

  PROBLEM-SOLVING MODEL:
  Science solves empirical and conceptual problems.
  Progress = solving more problems with fewer anomalies and inconsistencies.
  No reference to truth: progress is not convergence to truth.

  TYPES OF PROBLEMS:
  EMPIRICAL: natural phenomena that theory must explain.
  CONCEPTUAL: internal consistency, fit with other theories,
  methodological assumptions of the theory.

  CRITICISM OF INCOMMENSURABILITY:
  Laudan's "Reticulated Model": science involves multiple levels
  (aims, methods, theories) that constrain each other.
  Rational comparison of paradigms IS possible through shared
  second-order methodological values.

  CRITICISM OF REALISM:
  "Pessimistic meta-induction": the history of science contains
  many theories we now think are false (caloric, phlogiston, ether)
  that were empirically successful. Inference from past success to
  truth is therefore unreliable.
```

## Scientific Revolutions: Case Studies

### Copernican Revolution (1543)

```
  Ptolemy (geocentric): complex epicycles; predicted positions accurately.
  Copernicus: heliocentric; simpler explanatory structure;
  initially MORE complicated numerically.

  WHY ACCEPTED?
  Not empirical superiority (similar predictive accuracy).
  Simplicity (Copernicus), explanatory unification.
  Galileo's telescope observations (moons of Jupiter, phases of Venus).
  Newton's mechanics (retroactively): made heliocentrism explanatorily
  vastly superior.

  LESSON: Revolutions often involve accepting inferior theories
  (by current criteria) that open better research directions.
```

### Chemical Revolution (1770s-1790s)

```
  PHLOGISTON THEORY: Combustion releases phlogiston.
  Explained many facts; Priestley was brilliant phlogiston chemist.

  ANOMALY: Metals GAIN weight when burned (oxygen unites with them).
  Phlogiston with negative weight was proposed (ad hoc).

  LAVOISIER: Oxygen theory. Combustion = combination with oxygen.
  Initially: worse at explaining many things (e.g., why acids are acidic).
  Within a generation: phlogiston abandoned universally.

  WHY SO FAST?
  Lavoisier reformed chemical nomenclature (new language = new paradigm).
  A generation of young chemists adopted the new framework.
  Priestley and Cavendish (older) never converted.
  Kuhn: revolutions require the old guard to die out.
```

## Engineering Bridge: Paradigm Shifts in Platform Transitions

Kuhn's model is the most accurate description of how the software industry actually changes — not through gradual accumulation but through punctuated transitions where the old guard resists and the revolution is driven by newcomers:

**Normal science → anomalies → crisis → revolution** in software:

| Phase | Scientific Example | Platform Transition Example |
|-------|-------------------|----------------------------|
| Normal science | Ptolemaic astronomy; puzzle-solving within the paradigm | COBOL/mainframe: transactions, batch jobs, JCL scripts — all solvable within the paradigm |
| Anomalies accumulate | Orbital precession calculations don't quite fit | Web scale, user growth, 24/7 availability — mainframe paradigm strains but teams add auxiliary hypotheses (bigger boxes, more batch windows) |
| Crisis | Geocentric model requires ever more complex epicycles | The auxiliary hypotheses (add more RAM, tune the DB) stop working; incident rate rises; nobody knows how to scale further |
| Revolution | Copernican heliocentric model + Galileo's telescope | Client-server → web → cloud-native: the new paradigm doesn't just solve the old problems differently, it reframes which problems matter |
| Incommensurability | "Mass" means different things in Newtonian vs. Einsteinian physics | "Transaction" in a COBOL system vs. a distributed microservices system is not the same concept; ACID semantics do not translate to eventual consistency — the vocabularies are incommensurable |

**Major platform paradigm shifts (each incommensurable with the last):**
- Mainframe → client-server (1980s): terminal I/O → distributed state
- Client-server → web (1995-2000): fat client → stateless HTTP
- Web → cloud-native (2006-2015): pets → cattle; manual provisioning → IaC
- Cloud-native → AI-augmented (2020s): deterministic logic → probabilistic inference

**Kuhn's observation that revolutions are led by newcomers, not established practitioners**: every major platform transition saw established engineers resisting (client-server engineers who thought web was a toy; web engineers who thought mobile was peripheral; on-prem engineers who thought cloud was insecure). The paradigm entails tacit knowledge — decades of expertise that becomes a liability at the break.

**Incommensurability in breaking API changes**: a v1 API and a v2 API that share an endpoint name but have different semantics exhibit Kuhnian incommensurability. Translation is possible but imperfect — you can write an adapter, but you cannot write a lossless one. The semantic shift is real, not just syntactic.

## Common Confusion Points

**Kuhn did not say science is irrational.**
Kuhn argued that the standard (Popperian) model of scientific rationality is historically
inaccurate. He accepted that science makes progress. His point was that progress happens
through social and community processes, not through the individual falsificationist logic
Popper described.

**Incommensurability does not mean incomparability.**
Kuhn argued for partial incommensurability: comparison is possible but imperfect.
Full incommensurability (the extreme version) would make his own historical comparisons
impossible — which he acknowledged.

**"Paradigm" has proliferated unhelpfully.**
Masterman (1970) found 21 distinct senses in Kuhn's original text. The word is now used
loosely for "worldview" or "framework" in ordinary speech. In precision use, refer to
Kuhn's later "exemplar" (paradigm = model problem-solution) or "disciplinary matrix"
(paradigm = community's shared beliefs and techniques).

## Engineering Bridge: Lakatos's Research Programmes as Codebase Trajectories

Lakatos's model is more useful than Kuhn's for evaluating active engineering decisions, because it provides a prospective criterion (progressive vs. degenerating) rather than just a retrospective description of revolution.

```
  SCIENTIFIC RESEARCH PROGRAMME         CODEBASE / ARCHITECTURAL PROGRAMME
  ─────────────────────────────         ──────────────────────────────────
  Hard core                             Architectural invariants: the core
  (Newton's laws — not negotiable)      assumptions you will not abandon
                                        (e.g., "all state in Postgres,"
                                        "event-sourced writes," "REST-only API")

  Protective belt                       Adapter and shim layers: the code that
  (auxiliary hypotheses absorbing       absorbs anomalies without touching the
  anomalies)                            core (caching layers, retry wrappers,
                                        compatibility shims, feature flags)

  Positive heuristic                    The architectural roadmap: the intended
  (programme's research agenda)         direction for the next N quarters
```

**Progressive programme**: each modification to the protective belt predicts new capabilities that are then confirmed. The system handles more load, more complexity, with decreasing marginal cost per feature. Velocity stays high; the architecture compounds.

**Degenerating programme**: each modification is ad hoc. The shim layer grows; the core is more and more insulated from reality by workarounds. Velocity drops; each new feature requires understanding an increasingly baroque protective belt. The "technical debt" framing is exactly Lakatos's degenerating programme.

**The key Lakatosian insight for engineering decisions**: the question is not "does this modification fix the bug?" (that's too local) but "does this modification increase or decrease the programme's predictive scope?" A caching layer that merely hides a slow query is a degenerative move; a caching layer that enables a new class of real-time features is progressive.

**Lakatos's programmes are retrospective.**
You can only call a programme "progressive" or "degenerative" in retrospect.
At the time, you cannot always tell whether auxiliary hypothesis modifications are
progressive or ad hoc. This is Kuhn's counter-objection to Lakatos.
